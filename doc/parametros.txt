posición 1
Determina la acción que se va a realizar. Actualmente sólo se reconoce la acción 'eval'

posición 2
Nombre del problema que se va a evaluar.

** OPCIONES **

-f lista
Lista de archivos de código fuente que se evaluarán. Si un elemento de esta lista es una carpeta se expandirá para incluir los archivos dentro de ella (pero no otras carpetas, recursivamente). Se pueden usar los comodines propios de bash (*,?,{},[]) para listar más facilmente varios archivos.

-c lista
Lista de casos de prueba que se usarán para evaluar los códigos. Si un elemento de esta lista es una carpeta se expandirá para incluir los archivos de caso de prueba dentro de ella (pero no otras carpetas, recursivamente). Se pueden usar los comodines propios de bash (*,?,{},[]) para listar más facilmente varios casos, sin embargo puede ser que las expresiones no sean reconocidas como se espera, entonces rodee cada expresión en 'comillas simples'.

-o valor
Indica un nombre de archivo donde se guardarán los resultados de la evaluación en formato csv.
(actualmente el csv tiene nombre fijo)

-v
Indica que se mostrará información extra.

-nb
Indica que no debe mostrarse barra de progreso

** RESTRICCIONES **

-s valor
Tamaño máximo del archivo de código fuente en kiB.

-T valor
Tiempo máximo de compilación en ms.
(cambiar a segs)

-M valor
Memoria máxima de compilación en kiB
(cambiar a MiB)

-S valor
Tamaño máximo de la salida del programa del usuario en kiB
